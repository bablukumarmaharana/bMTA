# 🚀 bMTA – Open Source Bulk Mail Transfer Agent

bMTA is a **self-hosted, UI-driven bulk mailing platform** designed for high-volume email campaigns. It runs on a bare Linux server (no Docker) using Postfix, Dovecot, Apache, MySQL/MariaDB, and PHP. All management – domains, senders, bounce servers, DNS records, SMTP passwords – is done through a clean web dashboard.

## ✨ Features

- **Multi-domain & multi-sender** – Add unlimited domains and sender identities.
- **Automatic DNS record generation** – SPF, DKIM (2048-bit), DMARC, and MX records with copy buttons and live verification.
- **SMTP authentication** – Each sender acts as a full SMTP login (supports MailWizz, custom apps, etc.).
- **Email queue** – Database-driven queue processed by cron; supports HTML, plain text, AMP, and attachments.
- **Open & click tracking** – Transparent pixel and link rewriting with Apache rewrite rules.
- **Bounce handling** – IMAP-based bounce processing for multiple bounce servers.
- **Custom headers** – Unlimited per-sender and per-email headers.
- **Multi-user** – Admin and user roles; first admin created via web setup wizard (no hardcoded credentials).
- **Everything via UI** – No manual configuration files needed after installation.

## 📦 Requirements

- **Operating system:** Ubuntu 22.04/24.04 or Debian 12 (fresh install recommended)
- **Root access** (to install packages and configure services)
- **Domain(s)** you want to send from (DNS must be pointed to your server)

## 🚀 Quick Install

```bash
# 1. Create web root and clone the repository
sudo mkdir -p /var/www
cd /var/www
sudo rm -rf html
sudo git clone https://github.com/bablukumarmaharana/bMTA.git html
cd html

# 2. Run the automated installer
sudo bash install.sh
```

The installer will:
- Install Apache, MariaDB, PHP 8.x, Postfix, Dovecot, OpenDKIM
- Create the database and a random secure password
- Configure Postfix & Dovecot with MySQL virtual users
- Set up cron jobs for queue processing and bounce handling
- Enable Apache rewrite and set proper permissions

> 💡 **Optional:** Customize database credentials and application URL before running the installer:
> ```bash
> export BMTA_DB_NAME="bmta"
> export BMTA_DB_USER="bmta_user"
> export BMTA_DB_PASS="your_secure_password"
> export BMTA_BASE_URL="http://your-domain.com/"
> sudo -E bash install.sh
> ```

## 🌐 First Steps

1. Open your server’s IP or domain in a browser:  
   `http://<server-ip>/`
2. You’ll be redirected to the **admin account creation** page – set your email and password.
3. Log in and start configuring.

### Adding a Domain
- Go to **Domains** → add your sending domain.
- The dashboard will generate SPF, DKIM, DMARC, and MX records.
- Click **Copy** next to any value and paste it into your DNS provider.
- Use the **Verify Records** button to check DNS propagation.

### Creating a Sender
- Go to **Senders** → enter the local part (e.g., `newsletter`), select the domain, set an SMTP password.
- The bounce email is automatically taken from the bounce server (if selected).
- Each sender becomes a full SMTP login (email + password).

### Sending Emails
- Use the **Compose** page to queue a campaign (HTML, text, AMP, attachments, custom headers).
- The queue is processed every minute by a cron job.
- Monitor status under **Queue**.

### External SMTP Usage
- Use the **Dashboard** to see SMTP settings (host, port, encryption).
- Connect any application (MailWizz, custom scripts) using:
  - **Host:** your server IP
  - **Port:** 587 (STARTTLS) or 25
  - **Username:** full sender email (e.g., `newsletter@yourdomain.com`)
  - **Password:** the SMTP password set for that sender
- Manage sender passwords from the **Senders** table.

## 🛠 Configuration

All settings are in `/var/www/html/config/config.php`. They are auto-generated by the installer, but you can override them via environment variables:

| Variable          | Description                | Default               |
|-------------------|----------------------------|-----------------------|
| `BMTA_DB_HOST`    | Database host              | `localhost`           |
| `BMTA_DB_NAME`    | Database name              | `bmta`                |
| `BMTA_DB_USER`    | Database user              | `bmta`                |
| `BMTA_DB_PASS`    | Database password          | (randomly generated)  |
| `BMTA_BASE_URL`   | Application base URL       | `http://<server-ip>/` |

├── cron/                   # queue and bounce processing scripts

```

## 🔒 Security Notes

- **DKIM private keys** are stored in the database – never expose them publicly.
- Use **strong SMTP passwords** for senders (changeable any time in the UI).
- Keep your server and packages updated.
- Limit firewall to ports 80, 443, 587, 25 (and 993 for IMAP bounce retrieval).
- Use HTTPS in production (add an SSL certificate and update `BMTA_BASE_URL`).
